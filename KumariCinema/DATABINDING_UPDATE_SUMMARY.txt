======================================================
DATABASE TABLE STRUCTURE & DATABINDING UPDATE SUMMARY
======================================================

## DATABASE SCHEMA ALIGNMENT

All pages have been updated to match the actual database table structures provided. Below is the mapping of tables to pages:

### 1. CUSTOMER DETAILS (UserTable)
**Table:** UserTable
**Columns:** UserID, UserName, PhoneNumber, UserEmail, UserAddress
**Updates Made:**
- Removed: Email, ContactNumber, DateOfBirth, Gender, MembershipTier fields
- Kept: UserID, UserName, PhoneNumber, UserEmail, UserAddress
- Load Query: SELECT UserID, UserName, PhoneNumber, UserEmail, UserAddress FROM UserTable
- Form Fields: txtName, txtEmail, txtPhone, txtAddress

### 2. THEATRE DETAILS (Theatre)
**Table:** Theatre
**Columns:** TheatreID, TheatreName, City, TheatreAddress, TheatrePhoneNumber
**Updates Made:**
- Removed: TheatreCity, TheatreContactNumber, TheatreEmail, ParkingCapacity
- Kept: TheatreID, TheatreName, City, TheatreAddress, TheatrePhoneNumber
- Load Query: SELECT TheatreID, TheatreName, City, TheatreAddress, TheatrePhoneNumber FROM Theatre
- Form Fields: txtTheatreName, txtCity, txtAddress, txtContactNumber

### 3. HALL DETAILS (Hall)
**Table:** Hall
**Columns:** HallID, HallName, HallCapacity, HallType, ScreenSize, TheatreID (FK)
**Updates Made:**
- Removed: ExperienceType, Has3D fields
- Kept: HallID, HallName, HallCapacity, HallType, ScreenSize, TheatreID
- Load Query: SELECT H.HallID, H.HallName, H.HallCapacity, H.ScreenSize, T.TheatreName FROM Hall H INNER JOIN Theatre T
- Form Fields: txtHallName, txtCapacity, ddlScreenSize, ddlTheatre
- Note: Removed HallType dropdown if it was in original - kept only available fields

### 4. SHOWTIME DETAILS (Shows)
**Table:** Shows
**Columns:** ShowID, ShowDate, ShowTime, StartTime, EndTime, MovieID (FK), HallID (FK)
**Updates Made:**
- Kept: ShowID, ShowDate, ShowTime, MovieID, HallID
- Note: StartTime and EndTime fields exist but not used in simple form
- Load Query: SELECT S.ShowID, M.Title, S.ShowDate, S.ShowTime, H.HallName FROM Shows S...
- Form Fields: txtShowDate, ddlShowTime, ddlMovie, ddlHall

### 5. BOOKING DETAILS (Booking)
**Table:** Booking
**Columns:** BookingID, ConfirmationCode, ShowID (FK), PaymentID (FK), UserID (FK)
**Updates Made:**
- Removed: ConfirmationNumber, BookingTime, ReservationDate fields
- Kept: BookingID, ConfirmationCode, ShowID, UserID, PaymentID
- Load Query: SELECT B.BookingID, B.ConfirmationCode, U.UserName, M.Title, S.ShowDate FROM Booking B...
- Form Fields: ddlCustomer (UserTable), ddlShow (Shows), removed txtSeats, txtAmount

### 6. TICKET DETAILS (Ticket)
**Table:** Ticket
**Columns:** TicketID, TicketStatus, TicketNumber, SeatID (FK), CancellationID (FK), PriceID (FK), BookingID (FK)
**Updates Made:**
- Kept: TicketID, TicketNumber, TicketStatus, SeatID, BookingID
- Load Query: SELECT TicketID, TicketNumber, BookingID, TicketStatus FROM Ticket
- Form Fields: ddlBooking (ConfirmationCode), ddlSeat (SeatRow+SeatNumber), ddlStatus
- Load Seats: WHERE IsAvailable = 1 (numeric instead of 'Y')

### 7. SEAT DETAILS (Seat)
**Table:** Seat
**Columns:** SeatID, SeatRow, SeatNumber, IsAvailable
**Updates Made:**
- Removed: SeatCategory, HallID references
- Kept: SeatID, SeatRow, SeatNumber, IsAvailable
- Load Query: SELECT SeatID, SeatRow, SeatNumber, IsAvailable FROM Seat
- Form Fields: ddlRow, txtSeatNumber, chkAvailable
- IsAvailable: Changed from 'Y'/'N' to numeric 1/0

### 8. PAYMENT DETAILS (Payment)
**Table:** Payment
**Columns:** PaymentID, PaymentAmount, PaymentStatus, PaymentMode
**Updates Made:**
- Removed: PaymentMethod, PaymentDate fields
- Kept: PaymentID, PaymentAmount, PaymentStatus, PaymentMode
- Load Query: SELECT PaymentID, PaymentAmount, PaymentStatus, PaymentMode FROM Payment
- Form Fields: txtAmount, ddlStatus
- Note: PaymentMode field exists but not required in basic form

### 9. MOVIE DETAILS (Movie)
**Table:** Movie
**Columns:** MovieID, Title, Duration, Genre, Language, ReleaseDate
**Status:** ? ALREADY CORRECT (no changes needed)

### 10. DASHBOARD
**Updated to:** Use correct table references
- UserTable (not Users)
- Booking (ConfirmationCode not ConfirmationNumber)
- Shows table joins updated
- Payment table queries updated

## DATA TYPE CONVERSIONS

### IsAvailable Field Changes
- **Before:** Stored as VARCHAR2 with 'Y'/'N' values
- **After:** Stored as NUMBER with 1/0 values
- **Update Code:** `IsAvailable = 1` instead of `IsAvailable = 'Y'`

### Timestamp Fields
- **Removed:** OracleDbType.Timestamp (not available in OracleDbType)
- **Alternative:** Use OracleDbType.DateTime or OracleDbType.Date as needed
