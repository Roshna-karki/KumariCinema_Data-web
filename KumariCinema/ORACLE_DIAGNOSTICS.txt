======================================================
ORACLE CONNECTION DIAGNOSTIC GUIDE
======================================================

## Current Issue: ORA-00904 "DESCRIPTION": invalid identifier

This error suggests the Oracle TNS descriptor format is being parsed as SQL instead of being recognized as a connection string format.

## Diagnostic Checklist

### 1. VERIFY ORACLE DATABASE IS RUNNING

**Windows Command Prompt (Run as Administrator):**
```
sqlplus /nolog
SQL> conn cinema/1234@localhost:1522/ORCLPDB
```

**Expected Result:** 
- Connected to Oracle Database...
- SQL> prompt appears

**If Error Occurs:**
- Oracle service is not running
- Database is not accessible
- Hostname/port is incorrect

### 2. VERIFY CONNECTION STRING FORMAT

**Current Connection String:**
```
User Id=cinema;Password=1234;Data Source=(DESCRIPTION=(ADDRESS=(PROTOCOL=tcp)(HOST=localhost)(PORT=1522))(CONNECT_DATA=(SERVICE_NAME=ORCLPDB)));
```

**Alternative Formats to Try:**

**Option A: Easy Connect Format (Simplest)**
```
User Id=cinema;Password=1234;Data Source=localhost:1522/ORCLPDB;
```

**Option B: TNS Name Format (If tnsnames.ora is configured)**
```
User Id=cinema;Password=1234;Data Source=ORCLPDB;
```

**Option C: Full Connection String with semicolons**
```
User Id=cinema;Password=1234;Data Source=(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=localhost)(PORT=1522)))(CONNECT_DATA=(SERVICE_NAME=ORCLPDB)));
```

### 3. HOW TO TEST EACH FORMAT

**Step 1:** Open Web.config
```xml
<connectionStrings>
  <add name="OracleConnectionString" 
       connectionString="[PASTE FORMAT HERE]" 
       providerName="Oracle.ManagedDataAccess.Client" />
</connectionStrings>
```

**Step 2:** Restart the application

**Step 3:** Go to Movies page and check ~/Logs/MovieDetails_*.log

**Step 4:** Look for:
- "Connection opened successfully!" = Connection string is correct
- "ORA-00904" = Format needs adjustment
- Different error = Different issue

### 4. VERIFY ORACLE CLIENT CONFIGURATION

**Check ORACLE_HOME Environment Variable:**
```
Windows CMD: echo %ORACLE_HOME%
Should show: C:\app\[username]\product\[version]\client_1
```

**Check tnsnames.ora Location:**
```
Path: %ORACLE_HOME%\network\admin\tnsnames.ora
Or: C:\oracle\network\admin\tnsnames.ora
```

**tnsnames.ora Entry for ORCLPDB:**
```
ORCLPDB =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1522))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = ORCLPDB)
    )
  )
```

### 5. VERIFY ODP.NET INSTALLATION

**Check packages.config:**
```
Look for: Oracle.ManagedDataAccess
Version should be: 4.122.23.1 or higher
```

**If Missing or Outdated:**
```
NuGet Package Manager: Install Oracle.ManagedDataAccess
Or via Package Manager Console:
Install-Package Oracle.ManagedDataAccess -Version 4.122.23.1
```

### 6. VERIFY DATABASE USER CREDENTIALS

**Using SQL Developer or SQL*Plus:**
```
1. Open SQL Developer
2. Create new connection:
   - Connection Name: TestConnection
   - Username: cinema
   - Password: 1234
   - Hostname: localhost
   - Port: 1522
   - Service Name: ORCLPDB
3. Click "Test"
```

**If Test Fails:**
- Database user may not exist
- Password may be incorrect
- Service is not running

### 7. CREATE MISSING DATABASE USER (IF NEEDED)

**Connect as SYSTEM or SYSDBA:**
```sql
sqlplus sys/[sys_password] as sysdba
SQL> CREATE USER cinema IDENTIFIED BY 1234;
SQL> GRANT CONNECT, RESOURCE TO cinema;
SQL> GRANT UNLIMITED TABLESPACE TO cinema;
```

### 8. TEST CONNECTION PROGRAMMATICALLY

Create a test page (TestConnection.aspx):
```csharp
protected void Page_Load(object sender, EventArgs e)
{
    string connStr = ConfigurationManager.ConnectionStrings["OracleConnectionString"].ConnectionString;
    Response.Write($"Connection String: {connStr}<br>");
    
    try
    {
        using (OracleConnection conn = new OracleConnection(connStr))
        {
            conn.Open();
            Response.Write("? Connection successful!<br>");
            
            using (OracleCommand cmd = new OracleCommand("SELECT COUNT(*) FROM Movie", conn))
            {
                int count = (int)cmd.ExecuteScalar();
                Response.Write($"Movies count: {count}<br>");
            }
        }
    }
    catch (Exception ex)
    {
        Response.Write($"? Error: {ex.Message}<br>");
        Response.Write($"Exception Type: {ex.GetType().Name}<br>");
        Response.Write($"Stack Trace: {ex.StackTrace}<br>");
    }
}
```

### 9. CHECK APPLICATION LOGS

**Log File Location:**
```
C:\Users\asus\Desktop\KumariCinema\KumariCinema\Logs\MovieDetails_YYYYMMDD.log
```

**Look For:**
```
[timestamp] Connection opened successfully! = Good
[timestamp] ORA-00904 = Connection string format issue
[timestamp] ORA-12514 = Service name not found
[timestamp] ORA-12170 = Hostname/Port unreachable
```

## RECOMMENDED TROUBLESHOOTING STEPS

### Step 1: Start Simple (Easy Connect Format)
```xml
<add name="OracleConnectionString" 
     connectionString="User Id=cinema;Password=1234;Data Source=localhost:1522/ORCLPDB;" 
     providerName="Oracle.ManagedDataAccess.Client" />
```

### Step 2: If Step 1 Fails, Try TNS Format
```xml
<add name="OracleConnectionString" 
     connectionString="User Id=cinema;Password=1234;Data Source=ORCLPDB;" 
     providerName="Oracle.ManagedDataAccess.Client" />
```

### Step 3: Verify Connection Before Binding Data
Create TestConnection.aspx page from Step 8 above.

### Step 4: Check Logs for Detailed Error Messages
Review ~/Logs/MovieDetails_*.log for exact error details.

### Step 5: Verify Database User and Tablespaces
Use SQL Developer to ensure:
- User "cinema" exists
- User has CONNECT and RESOURCE roles
- User has UNLIMITED TABLESPACE
- Movie table exists

## COMMON ERRORS AND SOLUTIONS

| Error | Cause | Solution |
|-------|-------|----------|
| ORA-00904 | TNS format misunderstood | Use Easy Connect format |
| ORA-12514 | Service name not found | Verify service name in tnsnames.ora |
| ORA-12170 | Cannot reach hostname:port | Check Network connectivity |
| ORA-01005 | NULL password | Set password in connection string |
| ORA-00942 | Table doesn't exist | Run DATABASE_SETUP.sql |
| Login denied | Wrong credentials | Verify username/password |
