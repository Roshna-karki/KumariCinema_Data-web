WHAT EACH DESIGNER FILE DOES:

The .designer.cs files are auto-generated declarations that tell Visual Studio
about all the ASP.NET controls (TextBoxes, Buttons, DropDownLists, etc.) defined
in the corresponding .aspx files.

When a control is defined in the .aspx file like:
  <asp:TextBox ID="txtName" runat="server"></asp:TextBox>

The designer file declares it in C# as:
  protected global::System.Web.UI.WebControls.TextBox txtName;

This allows your .cs code-behind files to access these controls:
  txtName.Text = "John Doe";
  gvMovies.DataSource = movies;
  ddlGenre.SelectedValue = "Action";

Without proper designer declarations, Visual Studio can't find the controls,
causing compilation errors like:
  "CS0103: The name 'ddlCustomer' does not exist in the current context"
